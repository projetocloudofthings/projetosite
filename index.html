Habilitando o Paradigma de Nuvem das Coisas

As soluções propostas serão desenvolvidas como um conjunto de serviços,integrados em uma estrutura de software no nível de middleware, construído a partir de uma Arquiteturade Referência (RA) Middleware, serviços e RA concebidos especificamente para ambientes de CoT. Além disso, o projeto tem como objetivo Investigar e propor soluções para suplantar diversos desafios de pesquisa identificados no contexto de Cloud of Things (CoT) visando contribuir para permitir a plena realização desse paradigma.

Desenvolvimento das Seguintes Soluções Tecnológicas:

  • Modelo de virtualização de dispositivos
  • Especificação de serviços de alocação de recursos e escalonamento de tarefas
  • Especificação de serviços de segurança cibernética para CoT
  • Especificação de técnicas de gerenciamento de dados (armazenar, classificar e extrair conhecimento)
  • Uma arquitetura de referência para CoT
  • Especificação e implementação de um middleware para dar suporte à construção, implantação e execução de aplicações CoT
  • Um ambiente de programação seguindo uma abordagem dirigida a modelos para a especificação, verificação e implementação CoT.

Desafio 1: Virtualização de Dispositivos

Responder a duas principais questões de pesquisa:

  1. Como virtualizar dispositivos (sensores, atuadores e outros objetos físicos) e redes de dispositivos (RASSFs), e
  2. Como separar adequadamente as responsabilidades entre os componentes de uma Nuvem de Dispositivos (considerando as três camadas: de dispositivos, de gateways e da nuvem computacional)

Com relação a virtualização de dispositivos, além da proposta de novos modelos de virtualização de sensores, foi também proposto um modelo de virtualização talhado para sensores multimídia, os quais têm se mostrado extremamente relevantes e cada vez mais presentes em ambientes de IoT. O modelo de virtualização para sensores multimídia foi elaborado segundo um novo conceito, denominado de cadeias de serviços multimídia. Tais cadeias herdam características da área de virtualização de funções de redes, onde são criadas cadeias de funções virtuais (SFCs, do inglês Service Function Chains), as quais representam um serviço fim a fim para o usuário. Além do modelo em si, que foi tema de dissertação de mestrado de um aluno da UFF sob a orientação de duas pesquisadoras principais do projeto, foi elaborada uma linguagem de especificação para a criação de cadeias de serviços multimídia, descrita na Frente de Investigação 6 do projeto.

Além da proposta de novos modelos de virtualização de dispositivos, cujos resultados já foram reportados em relatórios anteriores, o projeto abordou a virtualização de redes, em particular, no contexto da abordagem emergente de Network Functions Virtualization (NFV) [4]. O paradigma de NFV é um dos pilares das redes 5G, as quais possuem metas bastante ambiciosas de provisão de serviços de diversas classes, para acomodar aplicações emergentes. Uma das classes de serviços que são alvo das tecnologias de 5G visa atender as aplicações da Internet das Coisas. Além disso, a integração dos paradigmas de NFV e Edge Computing tem sido extensivamente estudado e alavancado por órgãos de padronização como o ETSI. Nesse contexto, foi proposta uma arquitetura de referência para essa integração, no contexto de MEC (Multi-access Edge Computing), que inclui componentes para orquestração, virtualização e gerenciamento de recursos virtualizados. Nessa vertente do projeto, bem como na frente de investigação do desafio 2, foram propostas extensões a tal arquitetura de referência, com novos componentes voltados para a virtualização leve de dispositivos e para tarefas de alocação de recursos como o placement e o dimensionamento de funções virtuais. Como resultado, foram produzidos artefatos de software e publicações.

Desafio 2: Alocação de Recursos em Nuvens de dispositivos

Propor soluções para a concepção de dois processos:

  1. Provisionamento de recursos – decidir sobre qual nível da infraestrutura de CoT (físico, borda ou nuvem) e a quantidade/tipo de recursos serão engajados para a criação de VNs para atender as solicitações das aplicações;
  2. Um processo de escalonamento de tarefas para execução nos nós físicos que compõem um nó virtual, lidando com 2 requisitos potencialmente conflitantes: (i) satisfazer QoS das aplicações, (ii) gerar menor consumo dos recursos da infraestrutura CoT.

Desafio 3: Gerenciamento de Grandes Volumes de Dados

Propor mecanismos eficientes e escaláveis para a captura, processamento e armazenamento dos dados gerados pela infra de CoT.

Desafio 4: Segurança em CoT

Desenvolver um mecanismo de verificação de integridade de modo a aprimorar os métodos de verificação de integridade de software aplicáveis a sistemas embarcados de uma forma que seja imune a ataques de pré-processamento e também de replicação de código, bem como possa ser aplicado a diferentes plataformas de execução.

Desafio 5: Arquitetura de Referência para CoT

Para que os resultados das pesquisas derivadas dos desafios 1 a 4 sejam utilizados de forma efetiva na criação e no gerenciamento de sistemas de CoT, faz-se necessária a construção de um arcabouço de software provendo diversos tipos de funcionalidades para os vários componentes desses sistemas, e que forneça uma abstração única para uso pelos diversos tipos de stakeholders envolvidos.
Esse arcabouço será organizado como uma plataforma de middleware que proverá um conjunto de serviços e uma interface padrão para acesso ao sistema. Tal plataforma de middleware será baseada em uma Arquitetura de Referência (AR).

Desafio 6: Construção de aplicações para CoT

Criar um ambiente orientado por arquitetura para o desenvolvimento de aplicações para CoT. O ambiente será formado por dois níveis:

  1. Arquitetural, composto por uma linguagem de descrição de arquiteturas (ADL), com suporte a descrição estrutural, comportamental e de execução da arquitetura;
  2. Nível da plataforma de IoT, composto pelo arcabouço de software para dar suporte a nuvem de dispositivos que será construído no projeto
